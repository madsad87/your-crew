How Your Crew works

From request to reviewed work, every step stays visible.

  1. 01

    User gives a request

    A goal starts as plain language, not a hidden automation or brittle prompt chain.

  2. 02

    Orchestrator creates tasks

    The request becomes small Markdown task files with owners, dependencies, and acceptance criteria.

  3. 03

    Specialist agents complete work

    Builders, content creators, and other agents claim focused tasks and move completed work to review.

  4. 04

    Reviewer approves

    Review gates keep quality explicit before work is marked done or dependencies unlock.

  5. 05

    Workflow-runner continues the board

    The runner advances review and ready tasks in sequence until the board is clear or blocked.
