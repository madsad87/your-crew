# Dashboard UX Skill

## Purpose

Use this skill for dashboard usability, visual hierarchy, density, task-card clarity, responsive behavior, and read-only interaction design.

## Applies To

- ux-ui
- builder
- reviewer when validating dashboard UX changes

## Guidance

- Prioritize validation state and actionable work above historical context.
- Keep workflow colors semantic and restrained.
- Show metadata as compact chips or rows, not large panels.
- Preserve the task card status rail as the workflow signature.
- Avoid adding controls that imply task mutation unless explicitly requested.

## Out Of Scope

- New product navigation.
- Drag-and-drop task movement.
- Parallel execution controls.
- MCP/tool invocation UI.
