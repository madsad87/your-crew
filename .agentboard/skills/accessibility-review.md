# Accessibility Review Skill

## Purpose

Use this skill when reviewing or implementing interface changes that affect keyboard access, focus visibility, semantic controls, labels, contrast, or responsive readability.

## Applies To

- reviewer
- ux-ui
- builder when implementing interactive UI

## Guidance

- Preserve visible focus rings on interactive controls.
- Use semantic buttons only for actions and avoid nested buttons.
- Ensure non-interactive metadata chips do not look like controls.
- Keep labels readable in light and dark modes.
- Confirm long task titles, skill names, and file paths wrap instead of overflowing.

## Out Of Scope

- Automated accessibility tooling setup.
- External audit services.
- Browser extensions or MCP integrations.
